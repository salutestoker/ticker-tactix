# Web App QA Checklist

- App starts locally without errors.
- Primary page renders at desktop and mobile widths.
- Navigation works for all visible routes.
- Forms show validation, loading, success, and error states.
- Buttons and controls are keyboard accessible.
- Text does not overlap, truncate awkwardly, or overflow containers.
- Empty states and error states are handled.
- Console is free of relevant runtime errors.
- Network requests return expected status codes and payloads.
- Build, lint, typecheck, and tests pass where configured.
