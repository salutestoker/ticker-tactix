# Project Skills

Project-local Codex skills live here. Use this file as the skill index before opening any individual `SKILL.md`.

## How To Use

- Match the task to the smallest relevant skill below.
- Open only that skill's `SKILL.md`, then load referenced files only when needed.
- Do not bulk-load every skill or reference file into context.
- Add new skills as `.codex/skills/<skill-name>/SKILL.md`.

## Available Skills

| Skill | Use When |
| --- | --- |
| `laravel-specialist` | Laravel backend work: models, migrations, controllers, auth, queues, services, Eloquent, and PHP tests. |
| `laravel-inertia-react` | Inertia React pages, typed page props, `useForm`, shared props, layouts, routing, and Laravel/React integration. |
| `tailwind-design-system` | Tailwind CSS v4 tokens, reusable UI primitives, responsive layout, accessibility, and design-system consistency. |
| `shadcn` | Adding or adapting shadcn/ui source components, component composition, Radix-backed UI, and shadcn theming. |
| `typescript-advanced-types` | Complex TypeScript types, generics, utility types, inference, and compile-time type safety. |

## New Skill Checklist

- Use lowercase, numbers, and hyphens for the folder name.
- Keep the folder name and `name` field in `SKILL.md` aligned.
- Write a specific `description`; Codex uses it to decide when the skill applies.
- Keep `SKILL.md` concise and move detailed references into `references/`.
